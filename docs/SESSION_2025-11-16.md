## Lessons Learned

### 1. No Hardcoded Knowledge
**Lesson:** Hardcoded lists mislead agents and go stale  
**Solution:** Use patterns and discovery instead  
**Impact:** More reliable, transparent, maintainable

### 2. Documentation > Specs
**Lesson:** OpenAPI specs are often incomplete/outdated  
**Solution:** Crawl human-written documentation  
**Impact:** Much better quality and context

### 3. User Testing is Critical
**Lesson:** Found issues not caught in unit tests  
**Solution:** Comprehensive user validation  
**Impact:** Significantly improved tool quality

### 4. Pattern-Based Discovery Works
**Lesson:** Common patterns work for most APIs  
**Solution:** 20+ patterns + search fallback  
**Impact:** Works for unknown APIs dynamically
